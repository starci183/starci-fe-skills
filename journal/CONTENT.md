# Journal — learnings that compound over time

The **learning journal**: lessons this skill set picks up while it's used on a real app — design moves
that worked, gotchas hit, decisions and their rationale. Unlike [`../cannon/CONTENT.md`](../cannon/CONTENT.md)
(hard, prescriptive code rules), this is the **softer accumulated wisdom** — the "house gu" that grows from
feedback during `/starci-fe-ux-apply`, brainstorms, and reviews.

## How it grows (draft → merge, same shape as the cannon)
- Mid-work, a fresh lesson is staged as [`drafts/<name>.md`](drafts/).
- `/starci-fe-update-mindset` folds drafts into this file (**newest wins**), then deletes the draft.
- The skills READ this file so each run starts smarter — what got chosen, what to repeat, what to avoid.

## What belongs here vs the cannon
| | `cannon/CONTENT.md` | `journal/CONTENT.md` (this) |
|---|---|---|
| Kind | hard rule (RULE · example · VIOLATION) | soft lesson / taste / rationale |
| Voice | "you MUST / MUST NOT" | "we tend to / last time we chose…" |
| Source | scan of `src/` | feedback from using the skill |

> Starts mostly empty — it fills in as you **teach the skill by using it**. A general lesson that hardens
> into a real rule can be promoted into the cannon by the maintainer.
