# refs — course-website UX references (fetched LIVE, distilled over time)

We **don't keep static teardowns** anymore. When a brainstorm needs "how do the best learning sites solve
X", **fetch it fresh from the web** (always current) — then write the durable lesson here so next time is
faster. Same `drafts → CONTENT.md` shape as `../cannon/` and `../journal/`.

## How it works
1. A brainstorm needs a pattern (catalog · lesson player · gamification · onboarding · paywall · dashboard).
2. **Read this file first** for lessons already captured.
3. **Fetch live** (WebSearch/WebFetch) the relevant platforms for what's missing — current UX detail:
   Coursera, Udemy, Duolingo, Khan Academy, Brilliant, Codecademy, edX, Skillshare, Frontend Masters, freeCodeCamp.
4. Use it, then capture the durable takeaway into [`drafts/<topic>.md`](drafts/).
5. `/starci-fe-update-mindset` folds `drafts/` into this file (**newest wins**) → reused, less re-fetching.
